# File System

## 1. 파일 시스템

### 1) 디렉토리 와 파일

- **개요**:

- 리눅스는 유닉의 많은 부분을 그대로 사용
- 시스템 과 관련된 정보 와 하드웨어 같은 장치를 모두 파일로 관리
- 윈도우즈에서는 물리적인 디스크가 2개 있다면 디렉토리 구조도 2개이지만 리눅스에서는 언제나 시스템 전체에 하나의 트리만 가짐

- 파일의 종류

- 일반 파일
일반적인 파일
문서 파일(해석할 때 문자로 변경해서 해석) 과 바이너리 파일(바이트 코드 그대로 해석 -  실행 파일)로 구분

- 디렉토리 파일
해당 디렉토리에 저장된 파일이나 하위 디렉토리에 대한 정보를 소유
파일에 엑세스하기 위한 정보를 저장

- 링크 파일
시스템 사용자에게 편리성을 제공하기 위한 파일
하드 링크
    원본 파일을 복사해서 동일한 내용의 다른 사본 파일을 만드는 것
    원본 파일 과 링크 파일이 서로 다른 파일로 취급되기 때문에 하나가 삭제되도 나머지 하나는 그대로 남았음
    하나의 파일을 수정하면 연결된 다른 파일도 수정됩니다.
    파일에 대한 정보를 가진 i-node를 공유하기 때문에 가능

심볼릭 링크
    soft link 라고도 함
    바로가기 기능처럼 원본 파일의 위치를 기억해서 이 파일을 실행하면 원본 파일을 찾아서 실행하도록 만들어진 링크
    원본 파일이 삭제되면 동작하지 않습니다.

- 장치 파일
리눅스 시스템에 부착된 장치들을 관리하기 위한 특수 파일
/dev 디렉토리 아래에 위치

- 파일의 종류 확인: file 이라는 명령으로 확인이 가능
file .profile #일반 텍스트 파일

file /usr/bin/pwd #바이너리 파일 - 실행 파일

file /usr/bin #디렉토리

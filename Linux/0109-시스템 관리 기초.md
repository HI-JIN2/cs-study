# 시스템 관리 기초 (0109)

## 1. 작업 제어 및 예약

### 1) 작업 제어 (Job Control)
*   **Foreground**: 사용자가 명령을 입력하고 결과가 나올 때까지 기다리는 상태.
*   **Background**: 명령 뒤에 `&`를 붙여 실행. 프롬프트가 즉시 반환됨.
*   **작업 전환**:
    *   `CTRL + z`: 포그라운드 작업을 일시 중지(Stopped).
    *   `bg %작업번호`: 중지된 작업을 백그라운드에서 실행.
    *   `fg %작업번호`: 백그라운드 작업을 포그라운드로 전환.
    *   `jobs`: 현재 세션의 작업 목록 확인.
*   **nohup**: 로그아웃 후에도 프로세스가 계속 실행되도록 유지. (`nohup command &`)

### 2) 작업 예약
*   **at**: 정해진 시간에 **한 번만** 실행.
    *   `at [시간]`: 명령 입력 모드 진입 (종료 시 `CTRL + d`).
    *   `atq`: 대기 중인 작업 목록 확인.
    *   `atrm [작업번호]`: 작업 취소.
*   **cron**: **주기적**으로 반복 실행.
    *   `crontab -e`: 예약 작업 편집.
    *   `crontab -l`: 예약 작업 목록 확인.
    *   **형식**: `분 시 일 월 요일 명령어`
        *   `*`: 모든 값.
        *   `*/5`: 5분마다.
        *   예: `30 02 * * * backup.sh` (매일 새벽 2시 30분에 backup.sh 실행)

## 2. 소프트웨어 관리

### 1) APT (Advanced Package Tool)
*   우분투/데비안 계열의 패키지 관리 도구 (`.deb`).
*   **명령어**:
    *   `sudo apt update`: 패키지 목록 갱신 (설치 전 필수).
    *   `sudo apt upgrade`: 설치된 패키지 업그레이드.
    *   `sudo apt install [패키지명]`: 패키지 설치.
    *   `sudo apt remove [패키지명]`: 패키지 삭제 (설정 파일 유지).
    *   `sudo apt purge [패키지명]`: 패키지 및 설정 파일 완전 삭제.
    *   `apt search [키워드]`: 패키지 검색.

### 2) Snap
*   우분투에서 사용하는 샌드박스 형태의 패키지 관리 시스템(컨테이너와 유사).
*   의존성 문제를 해결하고 격리된 환경 제공.
*   `snap install [패키지명]`, `snap list`, `snap remove [패키지명]`.

### 3) 파일 압축 및 다운로드
*   **압축 (tar)**:
    *   `tar cvf file.tar dir`: 압축 (Create).
    *   `tar xvf file.tar`: 해제 (Extract).
    *   `tar zcvf file.tar.gz dir`: gzip으로 압축.
    *   `tar zxvf file.tar.gz`: gzip 해제.
*   **다운로드**:
    *   `wget [URL]`: 파일 다운로드.
    *   `curl -O [URL]`: 파일 다운로드 및 API 테스트.

## 3. 리눅스 부팅 프로세스

### 부팅 과정 개요
1.  **BIOS/UEFI**: 하드웨어 점검(POST), 부팅 장치 선택.
2.  **Boot Loader (GRUB)**: 커널 로드 및 부팅 옵션 선택.
3.  **Kernel Initialization**: 하드웨어 인식 및 드라이버 로드, 루트 파일 시스템 마운트.
4.  **Init Process (systemd)**: PID 1번 프로세스 실행. 시스템 서비스 및 데몬 실행.
5.  **Login Prompt**: 사용자 로그인 대기.

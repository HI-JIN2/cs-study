# 리눅스 쉘 및 파일시스템 (0107)

## 1. 편집기(Editor) 심화

### 1) 편집기 종류
*   **행 단위 편집기**: `ed`, `ex`, `sed` (스트림 편집기).
*   **화면 단위 편집기**: `vi` (표준), `vim` (vi 확장), `emacs` (Gnu 프로젝트), `nano` (초보자용).
*   **GUI 편집기**: `gedit` (그놈 데스크탑 기본).

### 2) Nano 편집기
*   비모드형 편집기 (입력 모드와 명령 모드가 분리되지 않음).
*   `CTRL` 키 조합으로 명령 수행.
*   **주요 단축키**:
    *   `CTRL + o`: 저장 (Write Out).
    *   `CTRL + x`: 종료 (Exit).
    *   `CTRL + w`: 검색 (Where Is).
    *   `CTRL + k`: 한 줄 잘라내기 (Cut Text).
    *   `CTRL + u`: 붙여넣기 (Uncut Text).

## 2. 쉘(Shell) 이란?

### 1) 개념
*   사용자의 명령을 커널에 전달하는 **명령어 해석기**.
*   Windows의 `cmd`나 `PowerShell`과 유사.
*   자체 프로그래밍 기능(**Shell Script**) 내장.

### 2) 쉘의 종류
*   **sh (Bourne Shell)**: 유닉스 초기 쉘.
*   **csh (C Shell)**: C언어 문법과 유사.
*   **ksh (Korn Shell)**: 유닉스 SVR4 표준.
*   **bash (Bourne Again Shell)**: 리눅스 표준 쉘. sh와 호환되며 기능 확장. (alias, history 등 지원)
*   **zsh**: 최근 많이 사용되는 확장 쉘 (Oh My Zsh).

### 3) 쉘 확인 및 변경
*   **현재 쉘 확인**: `echo $SHELL`
*   **사용 가능한 쉘 목록**: `cat /etc/shells`
*   **쉘 변경**: `chsh -s /bin/zsh 계정명` (로그아웃 후 적용)

## 3. 리눅스 파일 시스템(File System) 개요

### 1) 파일 시스템의 정의
*   파일이나 자료를 쉽게 발견하고 접근할 수 있도록 보관 또는 조직하는 체제.
*   리눅스는 모든 것을 **파일(File)**로 간주합니다. (하드웨어 장치, 프로세스 통신 등)

### 2) 리눅스 디렉토리 구조 (FHS: Filesystem Hierarchy Standard)
*   `/` (Root): 최상위 디렉토리.
*   `/bin`: **기본 명령어**(binary)가 위치. (ls, cp, mv 등)
*   `/boot`: **부팅** 관련 파일 (커널, 부트로더).
*   `/dev`: **장치 파일**(device)들이 위치. (터미널, 디스크 등)
*   `/etc`: 시스템 **설정 파일**(configuration)들이 위치. (passwd, network 등)
*   `/home`: **사용자 홈 디렉토리**. (root 제외)
*   `/lib`: 프로그램 실행에 필요한 공유 라이브러리.
*   `/media`, `/mnt`: 외부 장치(CD-ROM, USB) 마운트 포인트.
*   `/opt`: 추가 응용 프로그램 설치 디렉토리.
*   `/proc`: 커널 및 프로세스 정보를 담고 있는 가상 파일 시스템.
*   `/root`: **슈퍼 유저(Root)**의 홈 디렉토리.
*   `/run`: 시스템 부팅 이후의 가변 데이터 (PID 파일 등).
*   `/sbin`: 시스템 관리자용 명령어 (ifconfig, reboot 등).
*   `/tmp`: 임시 파일 저장소. (재부팅 시 삭제됨)
*   `/usr`: 사용자 유틸리티 및 애플리케이션 (Windows의 Program Files와 유사).
*   `/var`: 시스템 운영 중 발생하는 가변 데이터 (로그 `/var/log`, 메일 등).

### 3) 파일 경로
*   **절대 경로 (Absolute Path)**: 최상위 디렉토리(`/`)부터 시작하는 경로.
    *   예: `/home/user/document/file.txt`
*   **상대 경로 (Relative Path)**: 현재 위치를 기준으로 하는 경로.
    *   `.`: 현재 디렉토리.
    *   `..`: 상위 디렉토리.
    *   예: `../download/file.txt`

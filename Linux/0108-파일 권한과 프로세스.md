# 파일 권한과 프로세스 (0108)

## 1. 파일 시스템 명령어와 권한

### 1) 파일 및 디렉토리 관리
*   **pwd**: 현재 작업 디렉토리 확인.
*   **ls**: 디렉토리 목록 확인. (`-a`: 숨김파일 포함, `-l`: 상세 정보, `-F`: 파일 종류 표시)
*   **cd**: 디렉토리 이동. (`cd ~`: 홈으로, `cd ..`: 상위로, `cd -`: 이전 경로로)
*   **mkdir**: 디렉토리 생성. (`-p`: 하위 디렉토리까지 한 번에 생성)
*   **rmdir**: 빈 디렉토리 삭제.
*   **cp**: 파일 복사. (`-r`: 디렉토리 복사)
*   **mv**: 파일 이동 또는 이름 변경.
*   **rm**: 파일 삭제. (`-r`: 디렉토리 삭제, `-f`: 강제 삭제)
*   **touch**: 빈 파일 생성 또는 파일 시간 갱신.
*   **cat**: 파일 내용 출력.
*   **more / less**: 파일 내용 페이지 단위로 보기.
*   **head / tail**: 파일의 앞/뒤 부분 출력. (`tail -f`: 로그 실시간 모니터링)

### 2) 파일 권한 (Permission)
리눅스는 멀티 유저 시스템이므로 파일에 대한 접근 권한(읽기 `r`, 쓰기 `w`, 실행 `x`)을 소유자, 그룹, 기타 사용자로 나누어 관리합니다.

*   **권한 표기**: `drwxr-xr-x`
    *   첫 글자: `d` (디렉토리), `-` (파일), `l` (링크).
    *   나머지 9자리: 3자리씩 끊어서 소유자(u), 그룹(g), 기타(o) 권한.
*   **권한 변경 (chmod)**:
    *   숫자 모드: `chmod 755 file` (r=4, w=2, x=1)
    *   문자 모드: `chmod u+x file` (소유자에게 실행 권한 추가)
*   **소유자 변경 (chown)**:
    *   `chown user:group file`
*   **기본 권한 (umask)**:
    *   파일/디렉토리 생성 시 기본적으로 제외될 권한 설정.
    *   일반적으로 `022` (파일: 644, 디렉토리: 755).

### 3) 특수 권한
*   **SetUID (4000)**: 실행 시 파일 소유자의 권한으로 실행. (예: `passwd` 명령어)
*   **SetGID (2000)**: 실행 시 파일 소유 그룹의 권한으로 실행.
*   **Sticky Bit (1000)**: 디렉토리에 설정 시 누구나 생성 가능하지만 삭제는 소유자만 가능. (`/tmp` 디렉토리)

## 2. 프로세스 (Process) 관리

### 1) 프로세스란?
*   실행 중인 프로그램.
*   **PID (Process ID)**: 각 프로세스에 부여되는 고유 번호.
*   **PPID (Parent PID)**: 부모 프로세스의 ID.
*   **Daemon**: 백그라운드에서 실행되면서 서비스를 제공하는 프로세스.

### 2) 프로세스 확인 명령어
*   **ps**: 현재 실행 중인 프로세스 목록 확인.
    *   `ps -ef`: 모든 프로세스를 상세하게 출력 (System V 스타일).
    *   `ps aux`: 모든 프로세스를 사용자 중심으로 출력 (BSD 스타일).
*   **top / htop**: 시스템 상태 및 프로세스를 실시간으로 모니터링.
*   **pstree**: 프로세스 계층 구조를 트리 형태로 확인.

### 3) 프로세스 제어
*   **kill**: 프로세스 종료 시그널 전송.
    *   `kill [PID]`: 정상 종료 요청 (SIGTERM, 15).
    *   `kill -9 [PID]`: 강제 종료 (SIGKILL, 9).
*   **pkill**: 프로세스 이름으로 종료.
*   **killall**: 같은 이름을 가진 모든 프로세스 종료.

### 4) 백그라운드 작업
*   **&**: 명령어 뒤에 붙이면 백그라운드에서 실행.
*   **jobs**: 백그라운드 작업 목록 확인.
*   **fg**: 백그라운드 작업을 포그라운드로 가져옴.
*   **bg**: 정지된 작업을 백그라운드에서 실행 재개.
*   **nohup**: 터미널이 종료되어도 작업이 계속 실행되도록 함.
    *   `nohup command &`

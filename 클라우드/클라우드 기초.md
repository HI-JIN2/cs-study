IaaS -> PaaS -> SaaS -> FaaS

1. IaaS
	- 인프라
	- VM으로 가상화해서 서버(컴퓨터) 제공

2. PaaS 
	-  도커, 쿠버네티스
	- OS를 뺀 것. 근데 어플리케이션만 남은건 아니고 어플리케이션의 환경(라이브러리 등)까지 포함!
	-  그래서 기존 VM보다 가벼워졌다 
	-  동적으로 자원 적게 사용하면서 기존 기능 그래도 사용할 수 있음
	-  이미지화 해서 다른 쪽에서 수행하게 하기 쉬워짐 =이식성

3. SaaS
	- Software
	- MS 365
	- 내가 굳이 설치하지 않고도 쓸 수 있도록

4. FaaS
	- Function
	- Serveless =서버까지 관리하지 않아도 된다
	- 코드 소스만 올렸는데 기능 작동함! 
	- 서버를 생각해야했던 이유
		- 서버의 용량 때문 
		- 사용 안해도 띄워만 놔도 과금

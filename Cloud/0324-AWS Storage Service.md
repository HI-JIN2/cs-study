**AWS**

# 1. AWS Storage Service

## 1) 데이터 백업 및 보관
- **백업의 형태**:
  - 온프레미스(On-Premise) 환경의 데이터를 AWS로 백업
  - AWS 내에서 구축된 자원(EC2, RDS 등)을 다른 리전이나 계정으로 백업
- **백업 원칙**: 백업 대상(이미지, 데이터베이스, 로그 파일 등)을 정의하고 스케줄링하여 자동화

## 2) S3 (Simple Storage Service) 활용
- 높은 내구성과 가용성을 제공하는 객체 스토리지
- 백업 데이터의 보관 장소로 가장 널리 사용됨
- **버전 관리 (Versioning)**: 객체의 과거 상태를 유지하여 실수로 인한 삭제나 덮어쓰기 방지

## 3) EFS (Elastic File System)
- 여러 EC2 인스턴스에서 동시에 마운트하여 사용할 수 있는 확장 가능한 네트워크 파일 시스템 (NFS)
- 관리형 서비스로 별도의 서버 구성 없이 사용 가능

## 4) Storage Gateway
- 온프레미스 환경에서 AWS 스토리지로 연결하여 하이브리드 클라우드 스토리지 환경 구축 가능
- **File Gateway**: S3를 파일 시스템 인터페이스(NFS/SMB)로 노출
- **Volume Gateway**: 온프레미스 데이터를 EBS 스냅샷 형태로 백업

## 5) AWS Backup
- AWS 파이프라인 전반에 걸쳐 데이터 백업을 중앙 집중식으로 자동화하는 완전 관리형 서비스
- EC2, EBS, RDS, EFS, DynamoDB 등의 백업을 통합 관리 가능

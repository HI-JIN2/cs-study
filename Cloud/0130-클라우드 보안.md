# 클라우드 보안 및 네트워크 보안 (0130)

## 1. 보안의 개념과 3요소 (CIA)

### 1) 정보 보안 정의
위험이나 사고 없이 정보가 온전한 상태를 유지하도록 보호하는 모든 활동.

### 2) 보안의 3요소 (CIA Triad)
1.  **기밀성 (Confidentiality)**: 인가되지 않은 사용자가 정보를 볼 수 없도록 함. (예: 암호화)
2.  **무결성 (Integrity)**: 정보가 훼손되거나 변조되지 않고 정확성을 유지함. (예: 해시 함수 MD5, SHA를 통한 변경 확인)
3.  **가용성 (Availability)**: 인가된 사용자가 필요할 때 언제든지 정보에 접근할 수 있음. (DDos 방어 등)

## 2. 클라우드 자산 관리와 보호

### 1) 클라우드 자산의 특징
*   물리적 자산(CCTV, 건물 보안 등)은 클라우드 제공자에게 아웃소싱됨.
*   **불편한 이웃 문제**: 물리적 자원을 공유하므로 다른 사용자의 과부하가 내 성능에 영향을 줄 수 있음.
*   **하이퍼바이저/사이드 채널 공격**: 가상화 환경의 취약점을 이용한 공격 가능성 존재 (Spectre, Meltdown).

### 2) 자산 유형별 보안
*   **가상 머신 (VM)**: 어떤 OS/SW가 설치되었는지, 누가 접근 가능한지 목록화하여 관리.
*   **컨테이너**:
    *   호스트 OS 커널을 공유하므로 VM보다 공격 범위가 넓을 수 있음.
    *   **원칙**: 단일 기능 수행, 불변성 유지(업데이트 대신 재생성).
    *   **Orchestration (K8s)**: 클러스터 접근 제어 및 이미지 관리 중요.
*   **스토리지**:
    *   **Block Storage (EBS)**: 서버 OS 수준의 접근 통제.
    *   **Object Storage (S3)**: 버킷 및 객체별 ACL/정책 설정. (이미지에 민감 정보 저장 금지)
*   **Serverless**: 코드 취약점 관리 및 배포 목록 관리 집중.

## 3. ID와 접근 관리 (IAM)

### 1) 인증(Authentication) vs 인가(Authorization)
*   **인증**: "누구인가?" (ID/PW 확인)
*   **인가**: "무엇을 할 수 있는가?" (권한 부여, Access Control)

### 2) IAM 핵심
*   **ID 생명주기**: 생성 -> 사용 -> 재검증(퇴사자 처리 등) -> 삭제.
*   **MFA (Multi-Factor Authentication)**: '아는 것(PW)' + '가진 것(OTP/지문)'을 조합하여 보안 강화. 관리자 계정 필수 적용.
*   **Principle of Least Privilege**: 최소 권한 원칙. 필요한 만큼만 권한 부여.

### 3) SSO (Single Sign On)
*   **Federation ID**: 서로 다른 시스템 간 ID 연동.
*   **SAML / OIDC**: 중앙 ID 제공자를 통해 인증하고 티켓(토큰)을 받아 서비스 이용.

## 4. 네트워크 보안

### 1) 개념
네트워크 트래픽을 통제하여 시스템 공격 및 정보 유출 방지.

### 2) 보안 존(Zone) 구분
*   **Trust Network**: 내부의 보호받아야 할 네트워크.
*   **Untrust Network**: 신뢰할 수 없는 외부(인터넷).
*   **DMZ (De-Militarized Zone)**: 외부 서비스 제공을 위해 개방된 구간.

### 3) 보안 정책
*   **White List**: 허용할 통신만 명시 (나머지 모두 차단). 내부 방화벽에서 주로 사용.
*   **Black List**: 알려진 공격/문제 IP만 차단. IPS/WAF 등에서 사용.

### 4) 주요 보안 솔루션 (방어 계층 순서)

1.  **DDoS 방어 장비**:
    *   대용량 트래픽(Volumetric) 및 프로토콜 취약점 공격 방어.
    *   클라우드 기반 DDoS 방어 서비스 활용이 효과적.
2.  **방화벽 (Firewall)**:
    *   **L4** 계층 패킷 필터링 (IP/Port 기반 제어).
    *   가장 기본적이고 고성능.
3.  **IPS (Intrusion Prevention System)**:
    *   침입 방지 시스템. 시그니처(공격 패턴 DB) 기반으로 패킷 내용 검사 및 차단.
4.  **WAF (Web Application Firewall)**:
    *   **L7** 웹 애플리케이션 전용 방화벽.
    *   HTTP/HTTPS 공격(SQL Injection, XSS 등) 방어.
5.  **NAC (Network Access Control)**:
    *   단말기의 네트워크 접속 제어 (인가된 기기만 접속).

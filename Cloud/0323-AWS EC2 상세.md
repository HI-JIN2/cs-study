**AWS**

# 1. EC2 상세 및 로드밸런싱

## 1) EC2 (Amazon Elastic Compute Cloud) 상세
- Amazon EC2는 안전하고 크기 조정 가능한 컴퓨팅 용량을 제공하는 클라우드 서비스
- EC2는 매니지드 서비스(Managed Service)가 아니므로, OS 및 소프트웨어 레이어의 운영과 패치는 사용자가 직접 담당함

## 2) ELB (Elastic Load Balancing)
- 부하 분산 장치로, 들어오는 애플리케이션 트래픽을 여러 대상(EC2, 컨테이너, IP 등)으로 자동 배분함
- **ALB (Application Load Balancer)**: L7 계층(HTTP/HTTPS) 부하 분산. 경로 기반/호스트 기반 라우팅 지원
- **NLB (Network Load Balancer)**: L4 계층(TCP/UDP) 부하 분산. 매우 높은 성능과 고정 IP 제공
- **CLB (Classic Load Balancer)**: 이전 세대의 로드밸런서 (현재는 권장되지 않음)

## 3) 상시 가동 및 가용성 확보
- **Auto Scaling**: 수요 변화에 맞춰 EC2 인스턴스의 개수를 자동으로 조절
- **Multi-AZ**: 여러 가용 영역에 인스턴스를 배치하여 데이터 센터 장애 시에도 서비스가 중단되지 않도록 구성

## 4) 실습: 로드밸런서 구축
- 두 개의 가용 영역에 각각 하나씩 EC2 인스턴스 생성
- 타겟 그룹(Target Group) 생성 및 인스턴스 등록
- 로드밸런서(ALB) 생성 및 리스너(Listener) 설정
- 로드밸런서의 DNS 이름을 통해 접속 테스트

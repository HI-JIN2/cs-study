**AWS**

# 1. AWS 개요 및 VPC

## 1) AWS(Amazon Web Services) 개요
- 컴퓨팅, 스토리지, 데이터베이스, 데이터 분석, 네트워킹 등 다양한 IT 리소스를 제공하는 퍼블릭 클라우드 서비스
- 시스템 운영에 필요한 인프라 일체를 서비스 형태로 제공

## 2) VPC (Virtual Private Cloud)
- 사용자의 AWS 계정 전용 가상 네트워크
- AWS 클라우드 내에서 다른 가상 네트워크와 논리적으로 격리되어 있음
- IP 주소 범위 선택, 서브넷 생성, 라우트 테이블 및 네트워크 게이트웨이 구성 등 네트워크 환경을 완벽하게 제어 가능

### 주요 구성 요소
- **Subnet**: VPC의 IP 주소 범위를 나눈 부분 집합
  - **Public Subnet**: 인터넷 게이트웨이를 통해 외부 인터넷과 직접 통신 가능
  - **Private Subnet**: 외부 인터넷과 직접 연결되지 않으며, NAT 게이트웨이를 통해 외부로 나가는 통신만 가능
- **Internet Gateway (IGW)**: VPC와 인터넷 간의 통신을 가능하게 하는 관문
- **NAT Gateway**: 프라이빗 서브넷의 인스턴스가 인터넷에 접속할 수 있게 해주면서, 외부로부터의 원치 않는 접속은 차단
- **Route Table**: 서브넷 내부나 외부로 향하는 트래픽이 어디로 가야 하는지 결정하는 규칙 집합

## 3) 실습: VPC 환경 구축
- VPC 생성 -> 서브넷 생성 -> 인터넷 게이트웨이 생성 및 VPC 연결 -> 라우트 테이블 설정 및 서브넷 연결
- 보안 그룹(Security Group) 설정을 통해 인스턴스 수준의 트래픽 제어

**AWS**

# 1. AWS EC2 & EBS

## 1) EC2 (Elastic Compute Cloud)
- 클라우드에서 크기 조정이 가능한 컴퓨팅 용량을 제공하는 서비스
- 인스턴스를 통해 가상 서버를 생성 및 관리 가능

### 주요 용어
- **AMI (Amazon Machine Image)**: 인스턴스를 시작하는 데 필요한 소프트웨어 구성(운영체제, 애플리케이션 서버, 애플리케이션 등)을 포함한 템플릿
- **인스턴스 유형**: CPU, 메모리, 스토리지 및 네트워킹 용량의 다양한 조합 (예: `t2.micro`, `c5.large`)
- **Key Pair**: 인스턴스에 안전하게 로그인하기 위한 인증 수단 (공개 키는 AWS에 저장, 개인 키(`pem` 파일)는 사용자가 보관)
- **보안 그룹 (Security Group)**: 인스턴스에 대한 가상 방화벽 (포트 및 IP 허용 설정)

## 2) EBS (Elastic Block Store)
- EC2 인스턴스에서 사용할 수 있는 사용이 쉬운 고성능 블록 스토리지 서비스
- 범용 SSD, 전용 SSD, 하드 드라이브 등 다양한 볼륨 유형 제공

### 특징
- **지속성**: 인스턴스가 중지되어도 데이터 보존
- **스냅샷**: EBS 볼륨의 특정 시점 복사본을 만들어 S3에 저장 (백업 및 복구 용이)
- **동적 변경**: 서비스 중단 없이 크기나 유형 변경 가능

## 3) 실습: EC2 인스턴스 생성 및 EBS 마운트
```bash
# SSH 접속 예시
ssh -i "my-key.pem" ec2-user@ec2-1-2-3-4.compute-1.amazonaws.com
```
- 인스턴스 생성 후 EBS 볼륨을 추가로 생성하여 첨부(Attach)하고 운영체제 내에서 포맷 및 마운트하여 사용

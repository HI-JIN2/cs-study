**CI/CD**

# 1. CI/CD 환경 구축

## 1) 개요
- 애플리케이션 개발부터 배포까지의 전 과정을 자동화하여 효율을 높이는 방식
- **CI (Continuous Integration)**: 코드 변경 사항을 정기적으로 빌드 및 테스트하여 공유 레포지토리에 통합
- **CD (Continuous Deployment/Delivery)**: 통합된 코드를 실제 운영 환경에 자동으로 배포

## 2) 도커 이미지 빌드 및 푸시 자동화
- Jenkins를 사용하여 GitHub 소스 코드를 가져와 Docker 이미지를 빌드하고 Docker Hub에 푸시하는 과정

### 파이프라인 구성 요소
- **Checkout**: Git 소스 코드 가져오기
- **Build**: 애플리케이션 컴파일 및 빌드
- **Docker Build**: Docker 이미지 생성
- **Docker Push**: 생성된 이미지를 레지스트리에 저장

## 3) 쿠버네티스 배포 자동화
- 빌드된 이미지를 쿠버네티스 클러스터에 반영하는 과정

### 방식
- **Manifest 직접 수정**: `kubectl set image` 등을 사용하거나 매니페스트 파일의 태그를 변경하여 적용
- **Helm 사용**: Helm 차트를 업데이트하여 배포
- **ArgoCD 사용**: GitOps 방식으로 Git의 상태와 클러스터의 상태를 동기화

## 4) 실습: 전체 파이프라인 구성

```groovy
pipeline {
    agent any
    environment {
        DOCKER_IMAGE = "myuser/my-app"
    }
    stages {
        stage('Checkout') {
            steps {
                git branch: 'main', url: 'https://github.com/myuser/my-app.git'
            }
        }
        stage('Docker Build') {
            steps {
                sh "docker build -t ${DOCKER_IMAGE}:${env.BUILD_NUMBER} ."
                sh "docker tag ${DOCKER_IMAGE}:${env.BUILD_NUMBER} ${DOCKER_IMAGE}:latest"
            }
        }
        stage('Docker Push') {
            steps {
                script {
                    docker.withRegistry('', 'dockerhub-credentials-id') {
                        docker.image("${DOCKER_IMAGE}:${env.BUILD_NUMBER}").push()
                        docker.image("${DOCKER_IMAGE}:latest").push()
                    }
                }
            }
        }
        stage('Deploy to K8s') {
            steps {
                withKubeConfig([credentialsId: 'k8s-credentials-id']) {
                    sh "kubectl set image deployment/my-app my-container=${DOCKER_IMAGE}:${env.BUILD_NUMBER}"
                }
            }
        }
    }
}
```

## 5) 장애 대응 및 롤백
- 배포 후 문제가 발생할 경우 이전 버전으로 되돌리는 과정
- `kubectl rollout undo deployment/my-app`
- Jenkins 파이프라인의 `post` 블록에서 실패 시 알림(Slack 등) 설정

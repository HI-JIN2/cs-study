# 네트워크 분석 도구 (0123)

## 1. 네트워크 연결 확인

### 1) Ping (Packet Internet Groper)
*   **목적**: 대상 호스트와의 연결 상태 및 지연 시간 확인.
*   **프로토콜**: ICMP (Echo Request/Reply).
*   **사용법**:
    ```bash
    ping 8.8.8.8
    ping -c 4 google.com  # 4회 전송
    ```

### 2) Traceroute (Tracert)
*   **목적**: 목적지까지가는 경로(홉) 추적 및 구간별 지연 시간 확인.
*   **프로토콜**: UDP/ICMP (Linux), ICMP (Windows). TTL 값을 1씩 증가시키며 전송.
*   **사용법**:
    ```bash
    traceroute google.com (Linux)
    tracert google.com (Windows)
    ```

### 3) Telnet / TCPing
*   **목적**: 특정 포트(SERVICE)의 연결 가능 여부 확인 (방화벽 차단 확인).
*   **사용법**:
    ```bash
    telnet [IP] [PORT]
    # 접속 되면 화면이 바뀌거나 Connected 메시지 출력
    # 접속 실패 시 Connection refused 또는 Timeout
    ```

## 2. 네트워크 상태 및 정보 조회

### 1) Netstat / SS
*   **목적**: 현재 서버의 네트워크 연결 상태, 라우팅 테이블, 인터페이스 통계 확인.
*   **주요 옵션**: `-an` (모든 연결을 숫자로 표시), `-t` (TCP), `-u` (UDP), `-p` (프로세스 ID).
*   **사용법**:
    ```bash
    netstat -antp | grep LISTEN  # 현재 리슨 중인 포트 확인
    ss -antp                     # netstat보다 빠르고 상세한 정보 제공
    ```

### 2) Nslookup / Dig
*   **목적**: DNS 쿼리를 통해 도메인 정보 조회.
*   **사용법**:
    ```bash
    nslookup google.com
    dig google.com
    ```

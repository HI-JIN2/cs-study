## Q) 안드로이드에서 Serializable과 Parcelable의 차이점은 무엇이며, 일반적으로 컴포넌트 간 데이터 전달에 Parcelable이 선호되는 이유는 무엇인가요?

- 더 빠르고 안드로이드에 최적화 되어 있음
- 더 적은 가비지 객체를 생성하기 때문

## Pro Tips for Mastery: Parcel과 Parcelable이란 무엇인가요?


- Parcel은 고성능 프로세스 간 통신을 가능하게 하는 컨테이너 클래스
- Parcelable은 객체를 직렬화 해서 Parcel을 통해 전달할 수 있도록 하는 안드로이드 특화 인터페이스


# Q) 4. Context란 무엇이며 어떤 유형의 Context가 있나요?

- 앱의 상태, 환경에 대한 정보
- 리소스에 대한 접근 제공
- 종류
	- application context
		- 애플리케이션 생명주기와 연결
		- 전역적이고 오래지속
		- 라이브러리 초기화 
	- activity context
		- 액티비티 생명주기와 연결
		- 액티비티 리소스 접근, 다른 액티비티 호출
	- service context
	- 백그라운드 
	- broadcast context
		- 수명 짧음
	
## Pro Tips for Mastery: Context 사용 시 주의할 점은 무엇인가요? 

- 메모리 누수에 주의할 것
- 각 생명주기가 있기 때문에 그 생명주기가 끝났는데 그에 해당하는 context를 호출할시 메모리 누수, 크래시 발생, 비효율적인 리소스 처리 주의!!
	- 가비지 컬렉터가 해당 메모리를 회수 할 수 없게함 -> 메모리 누수
- 백그라운드 스레드에서 context 접근 xx
	- main스레드(=ui스레드)에서 해야함

## Pro Tips for Mastery: ContextWrapper란 무엇인가요? 🤔

- ContextWrapper = context를 상속하는 클래스
- context와 직접적인 소통을 하지 않고도 특정 기능 커스텀 가능
- context의 특정 동작을 개선하거나 재정의할때 사용
	- = context 동작을 커스텀
- 이점
	- 재사용성
	- 캡슐화
	- 호환성
## Pro Tips for Mastery: Activity에서 this와 baseContext 인스턴스의 차이점은? 🤔

- 둘다 context를 반환
- this
	- activity의 현재 컨텍스트 (고수준)
	- activity 자체를 참조
	- activity 생명주기와 관련된 작업
- baseContext
	- 원본 context를 참조해야 할 때 사용
	- 저수준의 context
	- 원본이 수정되지 않은 context에 대한 접근
